Getting Started with Static Analysis in C++

Static analysis is a powerful technique for finding bugs in your code before it even runs. In this post, I'll walk you through the fundamentals of static code analysis and introduce you to some amazing tools that can help improve your code quality.

What is Static Analysis?

Static analysis examines code without executing it, looking for potential bugs, security vulnerabilities, and code quality issues. Unlike dynamic analysis (which requires running the code), static analysis can catch problems early in the development cycle.

Getting Started with SVF

SVF (Static Value Flow) is an open-source framework for static analysis on LLVM. It's particularly powerful for pointer analysis and bug detection in C/C++ programs.

My Internship Journey at Splunk

Working at Splunk as a Backend Software Engineer was an incredible learning experience. I spent 16 weeks diving deep into build systems, workflow automation, and infrastructure at scale. Here's what I learned along the way.

The Build & Workflow Engine Team

I joined the Build & Workflow Engine team, responsible for maintaining and improving the CI/CD infrastructure that thousands of engineers depend on daily. The tech stack included Kubernetes, Argo Workflows, Bazel, and C++.

Key Takeaways

1. Scale Changes Everything
Building systems that work for 10 users is very different from building for 10,000. I learned to think about performance, reliability, and observability from day one.

2. Documentation is Code
Good documentation saves more time than clever code. I spent significant effort ensuring my changes were well-documented and easy for others to understand.

3. Testing is Non-Negotiable
In infrastructure work, thorough testing isn't optional. One mistake can break builds for hundreds of engineers, so we had comprehensive test coverage for everything.

Technical Challenges

Working with Bazel and Kubernetes presented unique challenges. Debugging distributed systems requires a different mindset - you need to think about failure modes, retries, and observability at every step.

Would I recommend an internship at Splunk? Absolutely! The team was supportive, the problems were interesting, and I grew significantly as an engineer. üåü

Building a Self-Sovereign Identity Wallet

StreetCred was one of the most exciting projects I've worked on - a mobile application that gives users complete control over their digital identity. Let me walk you through how we built it and the technologies that power it.

What is Self-Sovereign Identity?

Self-Sovereign Identity (SSI) is a model where individuals own and control their digital identities without relying on centralized authorities. Instead of Facebook or Google holding your data, you do - storing it securely on your device.

The Tech Stack

We built StreetCred using:

  • üì± React Native for cross-platform mobile development

  • üåê Decentralized Identifiers (DIDs) for identity management
  • üîó Verifiable Credentials for trusted data sharing
  • üé® Tailwind CSS + Figma for beautiful UI/UX

Cryptographic Challenges

One of the biggest challenges was implementing secure key management. Users needed to be able to prove their identity without revealing their private keys. We used:

  • Asymmetric encryption for secure communication
  • Zero-knowledge proofs for privacy-preserving verification
  • Secure enclaves for key storage on mobile devices

User Experience

The hardest part? Making complex cryptography feel simple. We spent weeks iterating on the UX, ensuring users could manage their identity without understanding the underlying technology. The goal was "magical simplicity" - it just works. ‚ú®

Self-sovereign identity is the future of digital identity management. As privacy concerns grow and users demand more control over their data, solutions like StreetCred will become increasingly important.
